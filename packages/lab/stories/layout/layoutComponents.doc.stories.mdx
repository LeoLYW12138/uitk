# Layout Components

Layout componentry was created to help you build consistent and responsive layouts while using Toolkit components.

## Border Layout

Defines the main content regions of an application, region or widget, such as a footer, header or side navigation.

The region is defined by the `position` prop.

  <Story name="Border Layout">{ToolkitBorderLayout.bind({})}</Story>
</Canvas>

## API


## Grid Layout

Defines an equally distributed layout system using columns and rows.

It defaults to a twelve column grid but you can customise it using the `columns` and `rows` props.

## Styling layout components

You can override a default style assignment in layout components, however, in some cases the styles are dynamically defined by JS values and overriding the CSS variable will not work.
